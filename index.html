Actividad 1 Base de Datos

1¿Qué es un Dato?

Un dato es una unidad básica de información que puede ser un número, una palabra, una fecha, o cualquier tipo de hecho que pueda ser registrado. Es lo más simple y elemental, como el nombre de una persona o una cifra de ventas.

2. ¿Qué es Información?

La información es un conjunto de datos procesados y organizados que tienen significado y utilidad para el usuario. Es el resultado de interpretar datos y presentarlos de manera comprensible y útil.

3. ¿Diferencia entre Dato e Información?

Dato: Son hechos aislados que no tienen significado por sí mismos.
Información: Son datos organizados y contextualizados que tienen significado y valor para quien los usa.

4. ¿Tipos de Datos?

Los tipos de datos definen qué tipo de valor puede almacenar un dato. Los más comunes son:

    Numéricos:INTEGER, FLOAT, DOUBLE
    Texto: CHAR, VARCHAR, TEXT
    Fecha y Hora: DATE, TIME, DATETIME
    Booleanos: BOOLEAN
    Binarios: BLOB, BYTEA
    Datos de Identificación: UUID

5. ¿Qué es un Modelado de Datos?

El modelado de datos es el proceso de crear una representación visual de cómo se organizarán los datos en una base de datos. Incluye definir las entidades, sus relaciones, y cómo se estructurarán los datos.

6. Tipos de Bases de Datos

Según Variabilidad:

    Estáticas: Los datos no cambian frecuentemente (ej. enciclopedias digitales).
    Dinámicas: Los datos cambian con frecuencia (ej. redes sociales, sistemas de gestión).

Según Contenido:

    Bases de Datos Transaccionales: Para operaciones diarias (ej. ventas, inventarios).
    Bases de Datos Analíticas: Para análisis de datos y reportes (ej. Data Warehouses).

Según Modelo de Base de Datos:

    Relacional: Datos organizados en tablas (ej. MySQL, PostgreSQL).
    NoSQL: Datos no organizados en tablas (ej. MongoDB, Redis).
    Orientada a Objetos: Datos como objetos (ej. db4o, ObjectDB).

7. ¿Qué es el Modelo Relacional?

El modelo relacional es un tipo de estructura de base de datos que organiza los datos en tablas (o relaciones) con filas y columnas. Cada tabla tiene filas (registros) y columnas (atributos), y las tablas pueden estar relacionadas entre sí mediante claves.

8. Elementos del Diagrama Entidad-Relacional

    Entidad: Representa un objeto o concepto (ej. Cliente, Producto).
    Atributo: Propiedad o característica de una entidad (ej. Nombre, Precio).
    Relación: Conexión entre entidades (ej. Compra).
    Clave Primaria: Identificador único para cada entidad (ej. ID del Cliente).
    Clave Foránea: Referencia a la clave primaria de otra entidad (ej. ID del Cliente en la tabla de Compras).

9. ¿Qué es una Entidad?

Una entidad es un objeto o concepto en el mundo real que tiene existencia y sobre el cual se quiere guardar información en la base de datos.

10. ¿Qué es un Atributo y sus Tipos?

Un atributo es una característica o propiedad de una entidad.

    Atributos Simples: No se pueden dividir en más partes (ej. Nombre, Edad).
    Atributos Compuestos: Se pueden dividir en sub-atributos (ej. Dirección: Calle, Ciudad, Estado).
    Atributos Derivados: Se calculan a partir de otros atributos (ej. Edad calculada a partir de la fecha de nacimiento).
    Atributos Multivaluados: Pueden tener varios valores (ej. Teléfonos de contacto).

11. ¿Qué es una Relación?

Una relación es una asociación entre dos o más entidades en una base de datos, que define cómo se conectan o interactúan entre sí.

12. Llaves Primarias y Foráneas

    Llave Primaria: Un atributo (o conjunto de atributos) que identifica de manera única a cada registro en una tabla.
    Características: Única, no nula, identificadora.
    Llave Foránea: Un atributo en una tabla que hace referencia a la llave primaria de otra tabla.
    Características: Establece relaciones entre tablas, puede ser nula o repetida.

Ejemplo:

    Llave Primaria: ID_Cliente en la tabla Clientes.
    Llave Foránea: ID_Cliente en la tabla Compras que se refiere a ID_Cliente en Clientes.

13. ¿Qué son las Reglas del Negocio?

Las reglas del negocio son las normas o políticas que definen cómo deben funcionar los procesos en una organización. Estas reglas se implementan en la base de datos para asegurar que los datos se manejen de acuerdo con los requisitos del negocio.

14. ¿Qué es la Cardinalidad o Mapeo?

La cardinalidad o mapeo describe la cantidad de instancias de una entidad que pueden estar relacionadas con una instancia de otra entidad. Se expresa en términos de relaciones 1:1, 1 , o N

    1:1 (Uno a Uno): Un cliente tiene una única cuenta.
    1 (Uno a Muchos): Un cliente puede hacer muchas compras.
    N (Muchos a Muchos): Un producto puede estar en muchas compras y una compra puede incluir muchos productos.
